# Tone Checker

- Check the markdown portion of a rule file (any `.md` file within `rules`) for tone, grammar, and spelling

## Desired Tone

- The main purpose is to be able to scan at the table to work out what the correct ruling is
  - Avoid replicating large quantities of potentially non-free content (e.g. referring to a table in a source book is acceptable)
- Bullet-point format, not prose — scannable over readable
  - An exception to the prose rule is when the impact of a rule is primarily narrative, in which case, a concise prose description to communicate the required mood is positive
- Short declarative sentences; imperatives for actions ("Take the Hide action")
- Key game terms bolded for quick reference
- Sections split by heading when a rule has multiple phases (e.g. "Staying Hidden", "Attacking
  While Hidden")
- No flavour text or narrative; mechanics only
- Opinionated summaries — captures the essential mechanic, not verbatim rule text

## Duplication

- Some level of duplication between notes is ok to allow for easy reference
- But large blocks of text shouldn't be duplicated across rules for a system
